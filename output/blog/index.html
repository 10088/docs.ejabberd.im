    Boxcar Android SDK 2.0.2

    Boxcar Framework v.2.0.2 makes the following changes:

    • Implement local tag subscription clean-up: On device registration, we now get the list of tags that are valid and list of tags that are deprecated. The SDK process the reply and only stores the tags that were reported as valid. On the other hand, the registration event informs which tags were signaled as ‘deprecated’ and ‘valid’ by the server.
    • PROGUARD processing: disable code optimization as this could case problems on ART devices

    The new SDK is available in developer download area.

    Using notification sounds on iOS and Android

    When implementing a push solution it is critical to think about all the small details that will make your app be more usable and stand out.

    We have often seen that customizing notification sound for iOS or Android apps is often simply ignored. When this happen, the application will use the default sound, which is often the same than the sound of SMS received.

    For your users, this is a great cause of confusion that lead to action to solve their issue:

    • They may disable sound just because they cannot know if this the alert they just had is a sound coming from your app or is a new message.
    • They may disable push totally, removing potentially important features to your application.
    • They may also decide that your app is annoying and uninstall the application.

    This article will help you implement custom sound for push notification in your application.

    Boxcar Android SDK 1.1.0.11

    Boxcar Framework v.1.1.0.11 makes the following changes:

    • Apache commons-codec removed as a dependency (isn’t needed anymore)
    • Add more retries for UDID service query
    • Allow developer to set username on each registration: do also discourage setting the username on framework initialization
    • Wake-lock protect ‘start’ method from boxcar api
    • Wake-lock for push callback from 3rd party application: ensure device can’t go to sleep while application callback is being executed to notify about an incoming push
    • Do use WakefulBroadcastReceiver for GCM: bypass GCMBroadcastReceiver class to replace it for a wakeful receiver. Use a custom implementation for GCMBaseIntentService that relies on the wake lock from the receiver instance instead of doing its own locking. This ensures the CPU from device can’t go to sleep while receiver is being executed.
    • Play no sound if file name is unknown or blank in payload
    • Allow developer to set a custom default sound
    • Use the sound file defined on payload: if filename is present on payload and the resource is available on ‘res/raw’ folder, then we play it instead of the default notification sound
    • Add 10.0.2.2 to the list of known hosts: this is the IP used by Android Emulator to access localhost

    The new SDK is available in developer download area.
